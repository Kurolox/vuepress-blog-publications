# And that's how I met your blog.

I've intended to make my own blog for quite a while, to be honest. I wanted to have a place where I could write about anything I'm working at the moment, and having a place where I can polish my english is also nice. However, I didn't just wanted to have a blog. I wanted to make a blog. And here you are.

Now, let's be honest. This blog is completely barebones right now. However, I didn't want to deploy a fully-fledged website from the get-go because that would take time. Instead, I want to slowly build upon this base and keep improving the site.

One of the things I wanted to do most was to write the articles using markdown. This would allow me to have some easy syntax for formatting the articles, support for code markdown, being able to write the articles however I wanted rather than being locked with some sort of browser-based WYSIWYG editor and, most importantly, having the articles being independant from the blog itself. That will allow me to modify the blog however I want without having to fear the loss of the articles I've written.

Now, regarding how this blog is made. This website is generated with Vuepress, a static site generator that uses Vue under the hood. I already have some experience with Vue (albeit only a bit), and being a static site generator means that I don't really have to worry about hosting. I could host it, don't get me wrong: Hosting is cheap and I've got an odroid C2 laying around doing nothing. However, I find the static approach to have more appeal since it allows me to also learn and practise another skill: Continuous Delivery.

You see, this blog is hosted in GitHub. I've got two repositories: One where I have the source of the blog with the articles written in markdown, and another one where the generated static files are deployed. I use Travis to automate the deployment, so all I have to do is push a new markdown file to the github repository and Travis deals with everything else.

I'd like to mention that I haven't made any of this from scratch. In fact, all of this has been done already by other people and has been explained throughly in this two articles I've used:

  • Creating a blog with Vuepress (by Adam Collier)
  • Publish a Vuepress site on GitHub pages with Travis (by Samuel Pouyt)

I'm slightly busy this days due to exams, and I just wanted a base to start writting. My goal is to expand this in the future: adding new features, changing styles and themes... using what I have right now as the foundation.
